# PETCIRCLE跨境工作台

这个仓库保存 PETCIRCLE跨境工作台和报表生成脚本的代码、规则文档、启动脚本和测试文件。

## 本仓库默认包含

- 工作台入口：`daily_ops_app.py`
- Temu / Shein / ERP 报表生成脚本
- 业务规则和需求文档
- 单元测试
- 启动和停止脚本

## 本仓库默认不包含

以下内容体积较大或包含业务数据，已通过 `.gitignore` 排除：

- `outputs/`
- `基础数据库/*.sqlite`
- 各平台数据源表文件夹
- 输入表、历史归档表
- 生成的 Excel / CSV / 安装包 / 运行缓存

如需迁移完整业务数据，请使用现有的迁移包脚本或单独传输数据目录。

## Python 依赖

```bash
pip install -r requirements.txt
```

## 桌面版启动

桌面版入口是 Electron 应用。首次启动前先检查依赖：

```bash
npm run doctor
```

检查通过后启动：

```bash
npm start
```

如果提示没有找到 Electron，请先执行：

```bash
npm install
```

Electron 下载可能较慢；如果安装长时间无进展，可以稍后重试。不要手动删除项目文件，尤其不要批量删除 `node_modules`、数据源或输出目录。

启动后也可以在桌面工作台右侧进入“系统设置”，点击“运行自检”。这个入口会检查 Electron、Python、关键文件、产品设计、真实渲染、界面按钮绑定、安全体检、模块体检、管理员/店长角色旅程、权限边界和核心业务测试，适合在换电脑、更新版本、批量导入前使用。

## 测试

桌面版核心业务测试：

```bash
npm test
```

这个命令会使用项目可用的 Python 环境，覆盖每日销量、导入缺失矩阵、任务包流转、任务屏蔽清单、销量差异对比、ERP 同步和店铺归属。不要直接用 `python3 -m unittest discover` 作为上线依据，它会跑到旧网页工作台和历史启动脚本测试。

## 交付检查命令

常用交付检查：

```bash
npm run check:ready
```

这个命令会依次检查启动环境、产品设计检查、真实渲染、界面绑定、安全体检、模块体检、角色旅程检查、核心业务测试、依赖安全和提交范围。它只检查，不会删除文件。

单项检查：

```bash
npm run check:design    # 产品设计检查：青春配色、响应式、角色化工作流、图片兜底
npm run check:render    # 真实渲染烟测：首屏、导航、角色切换、破图和横向溢出
npm run check:security  # 安全体检：Electron 隔离、IPC 暴露、权限边界、备份恢复白名单
npm run check:modules   # 模块体检：今日工作台、销量、任务、导入、报表、基础资料、ERP、备份
npm run check:journeys  # 角色旅程检查：管理员和店长关键路径是否顺畅
npm run check:docs      # 文档检查：README 是否覆盖当前交付检查和不删除文件原则
```

这些检查都不会批量删除文件，也不会移动业务数据。检查失败时先看提示原因，再只处理对应问题；不要为了让检查通过而删除本地数据源、输出表或图片素材。

## 提交前检查

提交到 git 或交付给他人前先运行：

```bash
npm run check:ready
```

这个命令会依次检查启动环境、产品设计、真实渲染、界面绑定、安全体检、模块体检、角色旅程、核心业务测试、依赖安全和提交范围。它只检查，不会删除文件。

如果 `check:ready` 显示“软件功能自检已通过，但提交范围检查未通过”，说明程序本身已经通过启动和业务测试，只是还有图片产物、打包产物或业务数据被 git 跟踪。此时不要删除本地文件，应先人工确认后，只从 git 提交范围移除这些产物。

如果只想单独检查提交范围，可以运行：

```bash
npm run check:release
```

提交范围检查会拦截仍被 git 跟踪的图片产物、打包产物、虚拟环境产物、业务数据源和输出表。若检查失败，需要先人工确认哪些文件只是产物，再从提交范围移除；不要批量删除本地文件。

## 全托任务工作台

工作台会继续沿用每周表格导入流程：店长可以补自己负责店铺的数据源，管理员可以补全局数据源并查看平台、店铺、批次、数据类型的缺失矩阵；系统生成报表后会自动把可处理明细写入“任务台账”。

任务状态流转：

```text
待推送 -> 待店长处理 -> 待管理员确认 -> 已完成
```

管理员按任务包确认推送；店长只看到自己负责店铺的数据，并按任务包整包填写处理动作、备注或凭证；管理员最后只需要确认完成，确认后任务从待办消失。主流程不做驳回。反复出现但暂不处理的 SKC/SPU/商家编码可以加入屏蔽清单，后续报表生成时不再重复提示。

桌面版顶部只有一个入口，通过“管理员 / 店长”视角切换控制数据范围：管理员看全部平台和店铺，店长必须填写姓名后才会看到自己负责的数据。一个店铺只归属一个平台，一个平台可以有多个店铺；默认内置 Temu、Shein、速卖通、TK、Ozon，管理员可以在“基础资料”里新增平台和店铺。

本机启动默认只允许本机访问：

```bash
python3 daily_ops_app.py
```

需要局域网内多人访问时，管理员电脑启动为局域网模式：

```bash
DAILY_OPS_HOST=0.0.0.0 DAILY_OPS_ADMIN_PASSWORD=你的管理员密码 DAILY_OPS_OWNER_PASSWORD=店长访问密码 python3 daily_ops_app.py
```

启动后，店长使用管理员电脑的局域网 IP 加端口 `8876` 访问。局域网模式下管理员登录必须输入 `DAILY_OPS_ADMIN_PASSWORD` 设置的密码；如设置了 `DAILY_OPS_OWNER_PASSWORD`，店长登录也必须输入该访问密码。本机模式仍可不设置密码，方便单机调试。数据文件、输出表、图片产物、打包产物和虚拟环境产物不要提交到 git；仓库只保留必要代码、规则和测试。

## 数据备份与恢复

管理员在桌面版“系统设置”里可以生成运营状态备份。备份文件会放到 `运营备份/`，内容只包含任务台账、规则、基础数据库和各类数据源，不包含 `outputs/`、图片产物、打包产物和虚拟环境产物。

恢复备份时需要粘贴 `.zip` 备份文件完整路径。恢复会覆盖当前运营状态和数据源，建议在每周大批量导入前、规则调整前、管理员审核大量任务前先生成一次备份。

## 每日和每周流程

每日主流程：

1. 店长进入“销量管理”，填写自己负责店铺的今日销售件数。
2. 系统对比昨天和 7 日均值，出现 50% 以上波动时提示原因，但不阻止提交。
3. 管理员在“今日工作台”查看未填、异常波动和待处理任务。
4. 店长进入“商品任务”，按任务包整包处理。
5. 管理员确认完成后，任务从待办消失。

每周辅助流程：

1. 店长或管理员在“数据导入”补齐自己负责店铺/全局需要的导入项。
2. 管理员查看缺失矩阵，确认哪些平台、店铺、数据类型还没有提交。
3. ERP 基础数据预留旺店通接口设置，第一版以手动同步为主，自动同步为可配置开关。
4. 系统把导入销量和店长每日填写销量做软对比，差异较大时提示管理员核对，不做硬拦截。

## 5 天版上线验收清单

管理员验收：

1. 使用管理员视角打开桌面工作台。
2. 在“系统设置”点击“运行自检”，确认显示自检通过。
3. 在“系统设置”先生成一次运营状态备份。
4. 在“基础资料”确认平台、店铺、负责人和每日填报开关。
5. 在“数据导入”上传 ERP、Temu、Shein、爆旺款、低分、议价等本周数据源，并查看缺失矩阵。
6. 点击生成就绪报表/任务，确认任务包中心出现待推送任务。
7. 管理员按任务包推送给店长。
8. 店长处理后，管理员点击确认完成，任务从待办消失。
9. 导出任务台账，确认 Excel 至少包含 `任务台账` 和 `操作记录` 两个页签。

店长验收：

1. 切换为店长视角，姓名必须和店铺负责人表中的负责人一致。
2. 进入“销量管理”，确认只能看到自己负责店铺的每日销量清单。
3. 填写今日销售件数，异常波动时补充原因。
4. 进入“数据导入”，确认只能看到自己负责店铺的导入缺口，并可以上传、结束上传或清空待提交文件。
5. 进入“商品任务”，确认只能看到自己负责店铺的任务包。
6. 点击“整包已处理”，填写处理动作和备注或凭证。
7. 提交后确认任务进入待管理员确认。
8. 点击“查看记录”，确认能看到店长提交时间、操作人、动作和备注。

权限验收：

1. 店长可以上传自己负责店铺的数据源，并必须先选择明确负责人。
2. 店长不能生成报表。
3. 店长不能保存规则。
4. 店长不能执行管理员审核。
5. 店长不能读取负责人全量配置和任务屏蔽清单。
6. 管理员可以推送、确认、导出、备份、恢复和运行系统自检。

## 提交仓库前检查

提交 git 前只保留必要代码、规则和测试。不要提交以下内容：

- `outputs/`
- `运营备份/`
- `基础数据库/*.sqlite`
- `基础数据库/operation_tasks.json`
- `temu数据源表/`、`shein数据源表/`、`erp数据源/`
- `核价输入表/`、`低分预警输入表/`
- 图片产物、打包产物、安装包、虚拟环境、运行缓存

当前目录如果出现大量同名文件同时显示删除和未跟踪，先不要提交，也不要批量删除；应先确认 git 工作区状态，再只提交必要代码文件。

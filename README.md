# 日常运营工作台

这个仓库保存日常运营工作台和报表生成脚本的代码、规则文档、启动脚本和测试文件。

## 本仓库默认包含

- 工作台入口：`daily_ops_app.py`
- Temu / Shein / ERP 报表生成脚本
- 业务规则和需求文档
- 单元测试
- 启动和停止脚本

## 本仓库默认不包含

以下内容体积较大或包含业务数据，已通过 `.gitignore` 排除：

- `outputs/`
- `基础数据库/*.sqlite`
- 各平台数据源表文件夹
- 输入表、历史归档表
- 生成的 Excel / CSV / 安装包 / 运行缓存

如需迁移完整业务数据，请使用现有的迁移包脚本或单独传输数据目录。

## 全托任务工作台

工作台会继续沿用每周表格导入流程：管理员上传 ERP、Temu、Shein、爆旺款、低分、议价等数据源，系统生成报表后会自动把可处理明细写入“任务台账”。

任务状态流转：

```text
待店长处理 -> 待管理员审核 -> 已通过 / 已驳回 / 已完成
```

店长可以按负责人筛选自己的店铺任务，填写处理动作和备注；管理员可以审核通过或驳回，并导出最终运营任务台账。

本机启动默认只允许本机访问：

```bash
python3 daily_ops_app.py
```

需要局域网内多人访问时，管理员电脑启动为局域网模式：

```bash
DAILY_OPS_HOST=0.0.0.0 python3 daily_ops_app.py
```

启动后，店长使用管理员电脑的局域网 IP 加端口 `8876` 访问。数据文件、输出表、图片产物、打包产物和虚拟环境产物不要提交到 git；仓库只保留必要代码、规则和测试。

## 数据备份与恢复

管理员登录后，在“输出文件”页面可以生成运营状态备份。备份文件会放到 `运营备份/`，内容只包含任务台账、规则、基础数据库和各类数据源，不包含 `outputs/`、图片产物、打包产物和虚拟环境产物。

恢复备份时需要粘贴 `.zip` 备份文件完整路径。恢复会覆盖当前运营状态和数据源，建议在每周大批量导入前、规则调整前、管理员审核大量任务前先生成一次备份。

## 5 天版上线验收清单

管理员验收：

1. 使用管理员身份登录工作台。
2. 在“输出文件”页面先生成一次运营状态备份。
3. 在“每周工作流”上传 ERP、Temu、Shein、爆旺款、低分、议价等本周数据源。
4. 点击“结束上传”，确认数据源卡片显示最新文件和记录数。
5. 点击“生成本周报表”，确认每份成功报表显示新增任务、更新任务和导入明细数量。
6. 进入“任务面板”，确认系统已生成爆旺冲突、低分预警、滞销处理、议价审核等任务。
7. 使用管理员身份审核一条店长已提交任务，确认可通过或驳回。
8. 导出任务台账，确认 Excel 至少包含 `任务台账` 和 `操作记录` 两个页签。

店长验收：

1. 使用店长身份登录，姓名必须和店铺负责人表中的负责人一致。
2. 进入“任务面板”，确认只能看到自己负责店铺的任务。
3. 点击“店长填写”，填写处理动作和备注。
4. 提交后确认任务状态进入 `待管理员审核`。
5. 点击“查看记录”，确认能看到店长提交时间、操作人、动作和备注。

权限验收：

1. 店长不能上传数据源。
2. 店长不能生成报表。
3. 店长不能保存规则。
4. 店长不能执行管理员审核。
5. 管理员可以审核、导出、备份和恢复。

## 提交仓库前检查

提交 git 前只保留必要代码、规则和测试。不要提交以下内容：

- `outputs/`
- `运营备份/`
- `基础数据库/*.sqlite`
- `基础数据库/operation_tasks.json`
- `temu数据源表/`、`shein数据源表/`、`erp数据源/`
- `核价输入表/`、`低分预警输入表/`
- 图片产物、打包产物、安装包、虚拟环境、运行缓存

当前目录如果出现大量同名文件同时显示删除和未跟踪，先不要提交，也不要批量删除；应先确认 git 工作区状态，再只提交必要代码文件。

# 基础资料与历史销量导入设计

## 目标

把 `店铺负责人对应表.xlsx` 和 `跨境运营总表new.xlsx` 写入系统，替代基础资料逐行录入。导入后系统可以生成店长账号、维护账号密码、按平台/店铺/时间查询历史销量，并让后续每日店长填报继续写入同一套销量台账。

## 范围

- 负责人导入：从负责人表提取平台、店铺、店名、负责人，写入现有店铺负责人配置。
- 店长账号：按负责人生成店长账号，保存密码摘要；管理员可重置密码。
- 历史销量导入：把运营总表的月度 sheet 拆成日期、平台、店铺、销量、来源信息。
- 查询报表：按平台、店铺、日期范围筛选，并返回总销量、日均销量、按平台汇总、按店铺汇总、每日明细。
- 月度备份提醒：管理员打开系统时，如果当月未做运营备份，提示导出备份。
- ERP 基础资料：保持接口同步为主，手工上传只作为兜底，不在本轮扩展。

## 数据设计

- `基础数据库/store_owner_map.json` 继续保存店铺负责人关系。
- `基础数据库/operator_accounts.json` 保存店长账号，字段包含角色、姓名、账号、密码摘要、盐、启用状态、更新时间。
- `基础数据库/daily_sales.json` 继续保存每日销量记录。本轮历史导入写入同一文件，并增加 `source`、`source_file`、`source_sheet`、`imported_at` 字段。
- 历史导入不会覆盖已有人工填写记录；同一天、同平台、同店铺已有记录时跳过，避免覆盖店长后续填报。

## 导入规则

负责人表使用 `店铺`、`店名`、`业务`、`平台` 四列。`业务` 为空的行跳过；平台为空时默认取表格平台列或 `Temu`。

运营总表只读取名称类似 `YYMM` 的月度 sheet，跳过 `总览` 和 `销量年度合计`。每个 sheet 第一行是店铺列，第一列是 Excel 日期序号或日期；`汇总` 列不导入。店铺平台优先用负责人配置匹配，匹配不到时按店铺名推断：带“弟”的店铺默认 `Temu`，`琪琪/童话/加加/宝宝/牛牛/二美/元元` 默认 `Shein`。已知店铺别名会先规范化，例如 `大额` 归为 `大鹅`，`美美` 归为 `大美`，再去匹配负责人配置。

## 界面设计

基础资料页增加“基础资料导入”区：

- 选择并导入负责人表。
- 选择并导入历史销量总表。
- 显示导入结果：负责人关系数、账号数、历史销量新增数、跳过数。
- 生成一份整理后的基础信息导入表到下载目录，便于人工复核和二次导入。

销量管理页增加“销量报表查询”：

- 日期开始、日期结束、平台、店铺筛选。
- 展示总销量、日均销量、记录数。
- 展示按平台、按店铺、按日期明细。
- 支持导出当前查询结果。

系统设置/今日工作台显示月度备份提醒：

- 管理员视角下，如果本月没有运营备份 zip，显示提醒。
- 点击现有“生成备份”即可消除提醒。

## 测试

- 单元测试覆盖负责人表解析、账号生成、密码重置、运营总表拆分、销量导入不覆盖人工记录、销量查询汇总。
- 桌面接口测试覆盖新增 CLI/API 调用。
- UI 自检覆盖新增按钮绑定。

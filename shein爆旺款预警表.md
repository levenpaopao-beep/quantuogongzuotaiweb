# SHEIN爆旺款重复预警表（正式版V2）

更新时间：2026-06-02

## 适用范围

本文件用于生成、复核和维护 SHEIN 平台爆旺款重复铺货预警表。

正式版V2 输出文件固定为 3 张表：

1. 总览
2. 具体店铺操作表
3. 说明

旧版的 `平销冲突明细表`、`爆旺款互相冲突明细表` 不再单独输出，冲突类型统一在 `具体店铺操作表` 内展示。

## 数据源

SHEIN 销售数据从项目目录下的 `shein数据源表` 文件夹读取：

- `20260601-shein琪琪.xlsx`
- `20260601-shein童话.xlsx`
- `20260601-shein加加.xlsx`
- `20260601-shein宝宝.xlsx`
- `20260601-shein牛牛.xlsx`

优先匹配 ERP 产品基础信息表；基础信息表匹配不到时，再匹配组合装基础信息表。

## SKU匹配规则

SHEIN 平台字段取 `商家SKU`。

匹配 ERP 时：

- 忽略 `@` 以及后面的字符。
- 只保留 `33` 开头，且以尺码后缀结尾的商家编码。
- 支持尺码顺序：`XS`、`S`、`M`、`MD`、`L`、`XL`、`XXL`。
- 与 ERP 的 `商家编码（新）` 匹配。
- 无法匹配 ERP 或组合装的 SKU，不进入预警明细。

示例：

| 原始商家SKU | 匹配用商家编码 |
|---|---|
| `330318716-XS@2` | `330318716-XS` |
| `330318716-MD` | `330318716-MD` |

## SHEIN取数字段

| 用途 | SHEIN字段 |
|---|---|
| 商品名称 | `供货方号` |
| SKC | `SKC` |
| 商家编码 | `商家SKU` |
| 申报价 | 优先 `供货价`；若无该字段，则取 `采购价` |
| 7天销量 | `近7天销量` |
| 30天销量 | `近30天销量` |
| 平台仓库存 | `SHEIN仓库存` |
| 上架天数 | `上架天数` |

平台仓库存必须按 SKU 维度展示，即每个 SKU 自己的库存数量，不使用 SKC 合计库存。

## 爆旺款定义

以 `店铺 + SKC` 为维度判断爆旺款。

新品：

- 上架天数 `< 30` 天
- 该 SKC 下所有已匹配 SKU 的 7天销量日均 `>= 10`

老品：

- 上架天数 `>= 30` 天
- 该 SKC 下所有已匹配 SKU 的 30天销量日均 `> 20`

## 重复铺货识别

同款按货品编码识别。

货品编码 = 商家编码去掉尺码后缀。

示例：

| 商家编码 | 货品编码 |
|---|---|
| `330318716-XS` | `330318716` |
| `330318716-S` | `330318716` |

同一货品编码出现在多个店铺或多个 SKC 中，且其中至少一个 SKC 是爆旺款，即进入重复铺货预警判断。

## 爆旺款优先级

当同款在不同店铺或相同店铺下均为爆旺款时，需要先判断保留优先级。

优先级规则：

1. 30天销量越高，优先级越高。
2. 30天销量相同时，申报价越高，优先级越高。
3. 30天销量和申报价都相同时，平台仓库存越多，优先级越高。

优先级高的爆旺款作为保护款；优先级低的同款进入操作表处理。

## 处理意见规则

| 场景 | 处理意见 |
|---|---|
| 平销款低于爆款报价 | `立即下架！` |
| 平销款不低于爆款报价，且重复款有平台仓库存 | `售完备货库存下架。` |
| 平销款不低于爆款报价，且重复款无平台仓库存 | `立即下架！` |
| 同为爆旺款但优先级低，且重复款有平台仓库存 | `禁止备货，30天内限时下架，清理库存。` |
| 同为爆旺款但优先级低，且重复款无平台仓库存 | `立即下架！` |

`立即下架！` 在表格中使用红字显示。

1. 琪琪

3. 加加
4. 宝宝

字段顺序：

| 顺序 | 字段 |
|---:|---|
| 1 | 店铺编号 |
| 2 | 店铺 |
| 3 | 负责人 |
| 4 | 爆款总数 |
| 5 | 重复铺货预计总数 |
| 6 | 平销冲突数 |
| 7 | 爆款互相冲突数 |
| 8 | 低于爆款报价数 |
| 9 | 不低于爆款报价数 |
| 10 | 立即下架数 |

说明：

- SHEIN 不使用数字店铺代码，`店铺编号` 字段也直接填写店铺名。
- 所有数量字段显示为整数，不保留小数点。

## 具体店铺操作表字段

字段顺序：

| 顺序 | 字段 | 说明 |
|---:|---|---|
| 1 | 商家编码 | 清洗后的商家编码，不含 `@` 后内容 |
| 2 | 货品名称 | ERP货品名称 |
| 3 | skc | 重复款 SKC |
| 4 | 所属店铺 | 重复款所属 SHEIN 店铺名 |
| 5 | 爆旺款skc | 被保护的爆旺款 SKC |
| 6 | 爆旺款店铺 | 被保护的爆旺款所属店铺 |
| 7 | 爆款报价 | 爆旺款对应 SKU 的申报价 |
| 8 | 重复款申报价 | 重复款对应 SKU 的申报价 |
| 9 | 爆款月销件数 | 爆旺款对应 SKU 的30天销量 |
| 10 | 重复款月销件数 | 重复款对应 SKU 的30天销量 |
| 11 | 是否低于爆款报价 | `是` 或 `否` |
| 12 | 爆款库存 | 爆旺款对应 SKU 的 SHEIN仓库存 |
| 13 | 重复款库存 | 重复款对应 SKU 的 SHEIN仓库存 |
| 14 | 负责人 | 重复款店铺负责人 |
| 15 | 冲突类型 | `平销款冲突爆款` 或 `多店铺均为爆旺款` |
| 16 | 处理意见 | 放在最后一列 |

字段要求：

- 删除旧版 `下架原因` 列。
- 表头必须使用简称：`爆款库存`、`重复款库存`，不要使用括号长表头。
- `是否低于爆款报价` 为 `是` 时标红底；为 `否` 时白底。
- `处理意见` 必须放在最后一列。

## 排序规则

`具体店铺操作表` 排序规则：

1. 先按 `skc` 升序排序。
2. 同一 `skc` 内，按商家编码尺码顺序排序。

尺码排序固定为：

`XS -> S -> M -> MD -> L -> XL -> XXL`

示例：

| skc | 商家编码 |
|---|---|
| `spxxxx` | `330318716-XS` |
| `spxxxx` | `330318716-S` |
| `spxxxx` | `330318716-M` |
| `spxxxx` | `330318716-MD` |
| `spxxxx` | `330318716-L` |
| `spxxxx` | `330318716-XL` |
| `spxxxx` | `330318716-XXL` |

## 数字格式

金额字段保留两位小数：

- `爆款报价`
- `重复款申报价`

数量字段显示整数，不保留小数点：

- `爆款月销件数`
- `重复款月销件数`
- `爆款库存`
- `重复款库存`
- 总览表内所有数量统计字段

说明表必须包含：

3. 表格结构

5. SKU口径

# Temu申报价异常V1

更新时间：2026-06-03

## 目标

生成 Temu 申报价异常交叉对比表，用于按店铺识别：

- Temu 申报价低于 ERP 成本价的亏损产品。
- Temu 申报价低于 ERP 批发价 80% 的破价产品。
- 各店铺在售 SKC、爆旺款、亏损 SKC、破价 SKC、7 天销量、30 天销量概况。

## 数据源

- Temu 在售数据：`temu数据源表/20260601-Temu仓库销售情况导出*.xlsx`
- Temu 爆旺款数据：`temu数据源表/Temu爆旺款.xlsx`，取该目录下最新版本。
- ERP 基础信息：`erp数据源/erp产品基础信息表*.xlsx`
- 店铺负责人：`店铺负责人对应表.xlsx`

## 字段映射

### Temu 字段

- 店铺：`店铺`
- SKC：`SKC`
- 商家编码 / SKU：`SKU货号`
- 申报价：`申报价格`
- 7 天销量：`7天销量`
- 30 天销量：`30天销量`

### ERP 字段

- 商家编码：`商家编码（新）`
- 货品名称：`货品名称`
- 货品规格：`规格名称`
- 成本价：`成本价`
- 批发价：`批发报价`

## 匹配规则

- Temu `SKU货号` 对应 ERP `商家编码（新）`。
- 若编码含 `@`，匹配时忽略 `@` 及后续字符。
- 输出明细中的 `商家编码` 使用 ERP `商家编码（新）`。
- 无法匹配 ERP 的行不进入异常明细，只在数据校验中统计。
- `申报价格 = 0` 视为停用或放弃产品，不进入异常明细。

## 预警定义

### 亏损产品

条件：

```text
Temu 申报价 < ERP 成本价
```

输出到 `低于成本价` 页签。

### 破价产品

条件：

```text
Temu 申报价 < ERP 批发价 * 80%
```

输出到 `低于批发价80%` 页签。

## 总览表口径

总览表按店铺输出以下字段：

- 店铺编码
- 店铺
- 负责人
- 在售 SKC 数量：Temu 在售导出中该店铺唯一 `SKC` 数量。
- 7 天销量：该店铺 Temu 在售导出中 `7天销量` 汇总。
- 30 天销量：该店铺 Temu 在售导出中 `30天销量` 汇总。
- 爆旺款数量：Temu 爆旺款表中该店铺唯一 `SKC` 数量。
- 亏损 SKC 数量：该店铺命中“低于成本价”的唯一 `SKC` 数量。
- 破价 SKC 数量：该店铺命中“低于批发价80%”的唯一 `SKC` 数量。

## 排序规则

异常明细按以下顺序排序：

1. 第一排序：`SKC`
2. 第二排序：`商家编码`
3. 商家编码尺码顺序：

```text
XXS -> XS -> S -> M -> MD -> L -> XL -> XXL
```

同一 SKC、同一款不同尺码时，按上述尺码顺序排列。

## 输出文件

命名规则：

```text
yymmdd-Temu申报价异常-版本号.xlsx
```

当前示例：`260603-Temu申报价异常-V1.xlsx`

输出页签：

- `总览表`
- `低于成本价`
- `低于批发价80%`
- `数据校验`
